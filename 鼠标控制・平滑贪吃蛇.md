## 📄 鼠标控制・平滑贪吃蛇

### 1. 项目概述

实现一款网页贪吃蛇游戏，具有以下核心特征：

- **鼠标控制蛇头方向**：蛇头实时朝向鼠标位置移动，支持任意方向（非传统上下左右）。
- **蛇身真实拖尾跟随**：每一节身体都严格跟随前一节的运动轨迹，间距恒定，视觉上形成连贯链条。
- **难度选择**：简单、中等、困难分别对应不同的移动速度（逻辑移动间隔）。
- **碰撞规则**：撞墙或撞自身（不包括无敌期间）立即游戏结束。
- **无敌保护**：游戏开始或重置后的前3秒内，蛇头无视边界和自身碰撞（且边界内不会飞出），并伴有视觉闪烁提示。
- **流畅动画**：逻辑移动与渲染分离，通过线性插值实现身体节段在两次逻辑移动之间的平滑过渡，消除“卡顿感”。

### 2. 技术栈与运行环境

- 纯原生 HTML/CSS/JavaScript，无需任何外部库。
- 使用 `<canvas>` 进行绘图，尺寸 1000×600 像素。
- 采用 `requestAnimationFrame` 驱动渲染循环，使用 `performance.now` 实现精确时间控制。

### 4. 核心算法与 Bug 应对方案

#### 4.1 鼠标控制方向计算

- **需求**：蛇头随时朝向鼠标位置移动。

- **实现**：

  javascript

  ```
  let dx = mouseX - head.x;
  let dy = mouseY - head.y;
  let mag = Math.hypot(dx, dy);
  let moveDir = mag < 0.001 ? lastDirection : { x: dx/mag, y: dy/mag };
  lastDirection = moveDir;
  ```

  

- **Bug 防范**：当鼠标与蛇头完全重合时（mag ≈ 0），使用上一次的有效方向，避免方向向量为零导致移动失效。

#### 4.2 蛇身移动与增长（真实拖尾）

- **需求**：每逻辑步长，蛇头沿方向前进固定距离 `MOVE_STEP`，身体各节跟随前节路径，长度在吃到食物后增加一节。

- **实现**：

  javascript

  ```
  // 新头位置
  let newHead = { x: head.x + moveDir.x * MOVE_STEP, y: head.y + moveDir.y * MOVE_STEP };
  // 构建新蛇：先将新头插入数组头部
  let newSnake = [newHead, ...logicSnake];
  if (!ate) newSnake.pop();   // 未吃到则移除尾部
  ```

  

- **Bug 防范**：不要手动逐个移动身体节点，利用 `unshift` + `pop` 自动实现链条跟随，避免位置错乱。
  此前遇到身体不变长或不同步的问题，根源在于错误地复制了旧身体或未正确更新数组长度。

#### 4.3 碰撞检测与无敌保护

- **碰撞种类**：边界碰撞（超出 canvas 半径范围）、自身碰撞（蛇头与身体其他节点距离小于半径和）。
- **无敌逻辑**：
  - 游戏开始/重置时设置 `invincibleEndTime = Date.now() + 3000`。
  - 碰撞检测前检查 `isInvincible()`，若为真则：
    - 边界碰撞改为**钳位**（将蛇头拉回边界内），不触发游戏结束。
    - 自身碰撞完全忽略。
  - 无敌期间蛇头闪烁（每 100ms 变换颜色）。
- **Bug 防范**：
  早期版本无敌结束后立即死亡，原因是在无敌期间错误地修改了蛇身坐标导致逻辑位置违和。解决方案：无敌时仅对渲染坐标进行钳位，逻辑坐标保持原样，并在下一次逻辑移动前强制同步边界内位置（但不致死）。

#### 4.4 难度切换：动态调整移动间隔

- 在 `setDifficulty` 中修改 `moveIntervalMs`，并重置计时基准 `lastMoveTimestamp` 和 `nextMoveTimestamp`，避免瞬间多次移动。
- 三个难度对应的间隔（毫秒）：简单 180，中等 125，困难 85。搭配 `MOVE_STEP=18` 可获得合理速度。

#### 4.5 平滑动画（消除卡顿）

- **问题**：早期版本只依赖 `setInterval` 或帧内直接移动，导致蛇身一跳一跳，不够流畅。

- **解决方案**：逻辑移动与渲染分离 + 线性插值。

  - 逻辑移动按固定时间间隔（如 125ms）执行一次，更新 `logicSnake`。

  - 每次逻辑移动后，根据当前鼠标方向**预计算**下一次移动后的蛇身位置，存入 `nextSnake`。

  - 渲染循环中，根据当前时间与上一次逻辑移动的时间差计算插值进度 `t ∈ [0,1]`，然后对每个身体节点进行线性插值：

    javascript

    ```
    let renderX = from.x + (to.x - from.x) * t;
    let renderY = from.y + (to.y - from.y) * t;
    ```

    

  - 使用 `requestAnimationFrame` 不断调用渲染函数，实现每帧平滑过渡。

- **注意**：食物位置不插值，始终显示实际坐标，不影响碰撞判定（判定仍基于逻辑坐标）。

#### 4.6 食物生成算法

- 必须避开蛇身所有节点，且不与蛇头重叠。
- 提供多重保障：随机尝试 3000 次 → 网格扫描 → 终极固定点。
- 每次吃到食物后立即重新生成，同时检查现有食物是否被蛇身意外覆盖（极少发生，但需兜底）。

### 5. 游戏循环架构（伪代码）

javascript

```
let lastMoveTime = 0, nextMoveTime = 0, moveInterval = 125;

function animationLoop(now) {
  requestAnimationFrame(animationLoop);
  if (游戏未结束且未暂停) {
    if (now >= nextMoveTime) {
      performLogicMove();        // 更新 logicSnake, 食物, 碰撞处理
      updateNextSnake();         // 预计算下一次移动目标位置
      lastMoveTime = now;
      nextMoveTime = now + moveInterval;
    }
    let t = (now - lastMoveTime) / moveInterval;
    let renderSnake = interpolateSnake(logicSnake, nextSnake, t);
    drawGame(renderSnake);
  } else {
    drawGame(logicSnake);        // 暂停/结束直接显示逻辑位置
  }
}
```

### 7. 常见 Bug 与最终解决方案汇总表

| 现象               | 原因                               | 解决方案                                                     |
| :----------------- | :--------------------------------- | :----------------------------------------------------------- |
| 蛇静止不动         | 移动逻辑未触发或方向向量为零       | 使用 `lastDirection` 缓存，确保每帧都有有效方向              |
| 蛇身不变长         | 移动算法错误（如手动移动节点）     | 采用 `unshift`+`pop` 并依据 `ate` 决定是否 `pop`             |
| 游戏一开始就结束   | 初始蛇身自身重叠或碰撞检测过于灵敏 | 初始身体间距设为 `MOVE_STEP`（18px），半径和 17px，留有安全余量；无敌保护默认开启 |
| 无敌结束后立刻死亡 | 无敌期间蛇头飞出边界未修正逻辑坐标 | 无敌时不致死但需钳位逻辑坐标至边界内（或在下一次移动前修正） |
| 运动卡顿不连贯     | 逻辑移动与渲染同频                 | 实施独立时间步长 + 线性插值渲染                              |
| 食物与蛇身重叠     | 随机生成未充分检测                 | 增加多次尝试与网格扫描保底                                   |
| 难度切换后速度突变 | 未重置时间基准                     | 同时重置 `lastMoveTime` 与 `nextMoveTime`                    |
