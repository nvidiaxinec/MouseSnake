# MouseSnake
MouseSnake – A smooth, mouse-driven snake game where you control the serpent’s direction freely. Follow your cursor, evade collisions, and grow longer with each bite. Features difficulty levels, invincibility protection, and fluid body animation.

## Source Code
```
请实现一个网页贪吃蛇游戏（HTML/CSS/JS，现代浏览器，自适应画布 1000×600），具体要求：

1. 核心玩法：鼠标控制蛇头方向（任意角度移动，非上下左右）。蛇头实时朝向鼠标光标移动，速度为固定像素步长（MOVE_STEP = 18px）。身体严格跟随头轨迹，每节间距等于步长。吃到食物身体增长一节，分数+1。

2. 碰撞与游戏结束：
   - 边界碰撞：蛇头超出画布（考虑半径9px）且无敌保护未开启时，游戏结束。
   - 自身碰撞：蛇头与身体任何一节距离小于半径之和（头半径9px，身体半径8px，阈值17px）且无敌保护未开启时，游戏结束。
   - 无敌保护：游戏开始或重置后的前3秒内，蛇头无视自身碰撞，边界碰撞改为钳位（将蛇头拉回边界内，不致死）。无敌期间蛇头闪烁（每100ms交替颜色），左上角显示剩余秒数。

3. 难度选择（简单、中等、困难）：
   - 通过改变逻辑移动间隔（ms）实现：简单 180ms，中等 125ms，困难 85ms。
   - 难度切换实时生效，无需重启游戏，切换后重置移动计时基准，避免速度突变。

4. 平滑动画（消除卡顿）：
   - 逻辑移动与渲染分离：按固定时间间隔（由难度决定）执行一次逻辑移动（更新蛇实际坐标、食物、碰撞）。
   - 每次逻辑移动后，根据当前鼠标方向预计算下一次逻辑移动后的蛇身位置（nextSnake）。
   - 使用 requestAnimationFrame 逐帧渲染，根据当前时间与上一次逻辑移动的时间差计算插值进度 t（0~1），对蛇身每个节点进行线性插值：`renderPos = logicPos + (nextPos - logicPos) * t`。
   - 食物位置无需插值，始终显示实际坐标。

5. 食物生成规则：
   - 必须完全避开蛇身所有节点（距离大于头半径+食物半径+2）。
   - 随机尝试最多3000次；若失败则进行网格扫描（步长30px）；终极后备固定在(120,120)（但保证不重叠）。
   - 每次食物被吃掉后立即重新生成；若因身体移动导致食物意外被覆盖，也立即重新生成。

6. 鼠标控制细节：
   - 监听 canvas 的 mousemove 事件，获取光标在 canvas 上的坐标（限制在边界内）。
   - 每次移动方向计算：从蛇头指向鼠标的单位向量。若鼠标与蛇头完全重合（向量长度<0.001），则使用上一次的有效方向（lastDirection），防止方向失效。
   - 画布上实时显示虚线引导线（从蛇头到鼠标），增强视觉反馈。

7. UI 与交互元素：
   - 画布下方提供：得分面板、难度按钮（3个高亮当前）、暂停/开始按钮、新的一局按钮。
   - 暂停时停止逻辑移动，渲染定格；游戏结束时显示遮罩和“新的一局”提示。
   - 新的一局重置所有状态（分数、蛇身、食物、无敌保护、移动计时基准）。

8. 绘图风格（可选但建议）：
   - 深色渐变背景 + 网格线。
   - 食物带光晕（红橙渐变）。
   - 蛇头带眼睛和信子，身体带斑点，使用圆形渐变增加立体感。
   - 无敌闪烁时蛇头变亮白色。

9. 技术实现要求：
   - 纯原生技术栈（HTML5/CSS3/JavaScript），无外部依赖。
   - 使用 canvas 并设置固定宽高 1000×600，CSS 保持实际尺寸。
   - 动画循环唯一入口：requestAnimationFrame，所有时间计算基于 performance.now。
   - 代码必须自包含在一个 HTML 文件中（<style> 和 <script> 内联），可直接复制保存为 .html 双击运行。

10. 必须避免的已知常见错误（Bug 防范）：
    - 蛇身不变长或拖尾断裂：采用 `unshift(newHead)` + (`!ate` 则 `pop()`) 方法，不要手动移动每个节点。
    - 游戏开始立即结束：初始蛇身间距设为 MOVE_STEP（18px），且无敌保护默认开启3秒。
    - 方向失效导致停止移动：保存 lastDirection，当鼠标坐标无效时复用。
    - 食物与蛇重叠：多重生成保底 + 覆盖检测。
    - 难度切换后速度混乱：同时重置 lastMoveTime 和 nextMoveTime。
    - 无敌期间飞出边界导致逻辑错乱：只对渲染坐标钳位，逻辑坐标维持原状，但确保下次移动前不会死亡（使用钳位修改逻辑坐标或标志位处理）。建议：无敌时碰撞检测返回值不变，但移动后的新蛇不触发 gameOver，且对飞出边界的蛇头进行坐标修正（设回边界内，但不结束）。
    - 暂停/恢复后瞬间多次移动：恢复时重置时间基准。

11. 额外提供命令行友好性（可选）：代码生成后可以直接运行，无需额外配置。

输出要求：完整的 HTML 文件，包含 DOCTYPE、样式、脚本，确保移动端触摸友好（但核心测试使用鼠标）。最终提供代码。

```

项目核心思想：使用一段合理的提示词，使得任意具有编程能力的ai可以一次完成该任务。同时，这样的完整编程提示词也可以作为一种ai能力评价指标。
